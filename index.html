What we learned building fast.

RAG over fine-tuning Card data changes. Retrieval keeps answers grounded in current data without retraining.
Proxy everything Browser → frontend → backend. Never expose the backend directly to the client.
UX first, features second Hiding model controls from users was a deliberate call — complexity belongs in the backend.
Data quality matters Manually scraped card data is imperfect. Good prompting can compensate, but only so far.
